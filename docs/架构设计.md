# 架构设计

## 1. 技术选型决策

### 1.1 运行时：Bun

| 维度 | Bun | Node.js |
|------|-----|---------|
| 内存（idle） | **20-50 MB** | 50-80 MB |
| 内置 SQLite | `bun:sqlite`（快 3-6x） | 需 better-sqlite3 |
| 原生 TypeScript | 直接运行 .ts/.tsx | 需 tsx / ts-node |
| 包管理 | 内置（比 npm 快 25x） | npm / pnpm |
| 启动速度 | ~50ms | ~200ms |

### 1.2 框架：Hono

| 维度 | Hono | Elysia | Bun.serve 裸用 |
|------|------|--------|----------------|
| SSR/HTML | **内置 JSX** | 需装插件 | 手写字符串 |
| 中间件生态 | **50+ 官方** | 较少 | 无 |
| 稳定性 | **4.x 无 breaking** | patch 也有 breaking | - |
| 运行时锁定 | 不锁定 | 强绑 Bun | 强绑 Bun |
| 周下载量 | **950 万** | 25 万 | - |

### 1.3 ORM：Drizzle

数据库可插拔的关键。同一份 schema 定义，切换数据库只改连接配置：

```typescript
// 同一套 schema 定义，三种数据库通用
import { sqliteTable, text, integer } from "drizzle-orm/sqlite-core";
// 切换时改为:
// import { mysqlTable, varchar, int } from "drizzle-orm/mysql-core";
// import { pgTable, varchar, integer } from "drizzle-orm/pg-core";
```

Drizzle 的优势：
- SQL-like API，学习成本低
- 零运行时开销（编译期生成 SQL）
- 同时支持 `bun:sqlite`、`mysql2`、`postgres`
- TypeScript 类型推断到字段级别

### 1.4 双语言策略

```
┌─────────────────────────────┐  ┌─────────────────────────┐
│    Registry 服务端           │  │    Gradle 发布插件       │
```

两个项目完全独立，通过 HTTP API 交互。Gradle 插件写好后基本不动，200 行 Kotlin 代码。

## 3. 模块职责

### 3.1 Routes 层

三组独立的 Hono 路由，通过 `app.route()` 挂载：

```typescript
// src/index.ts
import { Hono } from "hono";
import { logger } from "hono/logger";
import { config } from "./config";
import { createRepository } from "./repository";
import { PluginParser } from "./service/plugin-parser";
import { PluginService } from "./service/plugin-service";
import { createRepositoryRoutes } from "./routes/repository";
import { createApiRoutes } from "./routes/api";
import { createWebRoutes } from "./routes/web";
import { mkdir } from "fs/promises";

// 启动校验
if (!config.adminPass) {
  console.warn("WARNING: ADMIN_PASS is not set. Web management login will be disabled.");
}
if (config.authTokens.length === 0) {
  console.warn("WARNING: AUTH_TOKENS is not set. API upload/delete will be blocked.");
}

// 初始化数据目录
await mkdir(`${config.dataDir}/plugins`, { recursive: true });
await mkdir(`${config.dataDir}/tmp`, { recursive: true });

// 初始化各层
const repository = createRepository();
repository.initialize();

const parser = new PluginParser();
const service = new PluginService(repository, parser, config.dataDir, config.baseUrl);

// 组装 Hono 应用
const app = new Hono();

// 全局中间件
app.use("*", logger());

// 挂载路由
app.route("/", createRepositoryRoutes(service, config));   // /updatePlugins.xml, /plugins/*
app.route("/api", createApiRoutes(service, config));       // /api/plugins, /api/health, /api/stats
app.route("/", createWebRoutes(service, config));          // /, /web/*

console.log(`Plugin Registry started on port ${config.port}`);
console.log(`  Base URL: ${config.baseUrl}`);
console.log(`  Data dir: ${config.dataDir}`);
console.log(`  Database: ${config.db.type}`);

export default {
  port: config.port,
  fetch: app.fetch,
};
```

### 3.2 Service 层 (`PluginService`)

单一 Service 类，协调所有业务逻辑：

```typescript
class PluginService {
  private startedAt = new Date();

  constructor(
    private repository: PluginRepository,  // 数据库操作
    private parser: PluginParser,          // ZIP 解析
    private storageDir: string,            // 文件存储目录
    private baseUrl: string                // 服务公网地址
  ) {}

  async upload(file: File, force: boolean): Promise<PluginVersion> { ... }
  async deleteVersion(pluginId: string, version: string): Promise<number> { ... }
  async deletePlugin(pluginId: string): Promise<void> { ... }
  getPlugin(pluginId: string): PluginWithVersions | null { ... }
  getUpdatePluginsXml(buildNumber?: string): string { ... }
  getPluginFilePath(pluginId: string, version: string): string { ... }
  listPlugins(query: string | null, page: number, pageSize: number): PagedResult<PluginSummary> { ... }
  getStats(): RegistryStats { ... }
  getRecentUploads(limit: number): RecentUpload[] { ... }

  /** 返回人类可读的运行时间，如 "3d 12h 30m" */
  getUptime(): string {
    const ms = Date.now() - this.startedAt.getTime();
    const d = Math.floor(ms / 86400000);
    const h = Math.floor((ms % 86400000) / 3600000);
    const m = Math.floor((ms % 3600000) / 60000);
    return `${d}d ${h}h ${m}m`;
  }

  /** 返回服务启动时间的 ISO 字符串 */
  getStartTime(): string {
    return this.startedAt.toISOString();
  }
}
```

### 3.3 Repository 层（可插拔数据库）

详见 [数据模型.md](数据模型.md)

接口定义 `PluginRepository`，默认实现 `SqlitePluginRepository`（基于 Drizzle + bun:sqlite）。切换数据库只需：
1. 安装对应驱动（`mysql2` 或 `postgres`）
2. 修改环境变量 `DB_TYPE` 和 `DB_URL`

### 3.4 Parser 层 (`PluginParser`)

详见 [插件解析器.md](插件解析器.md)

从上传的 ZIP/JAR 中提取 `plugin.xml` 并解析为结构化的 `PluginMetadata`。

### 3.5 Views 层（JSX 组件）

详见 [Web管理界面.md](Web管理界面.md)

使用 Hono 内置 JSX，`.tsx` 文件直接写 HTML 组件，无需额外模板引擎。

## 5. 配置体系

### 5.1 环境变量

所有配置通过环境变量，Bun 原生支持 `.env` 文件：

```bash
# .env
PORT=3000
BASE_URL=https://plugins.example.com
AUTH_TOKENS=token1,token2
DATA_DIR=./data

# Web 管理界面认证
ADMIN_USER=admin              # 管理员用户名，默认 admin
ADMIN_PASS=your-password      # 管理员密码，必填
SESSION_SECRET=random-string  # cookie 签名密钥，默认自动生成

# 数据库（默认 SQLite）
DB_TYPE=sqlite
# SQLite: 文件名（相对 DATA_DIR）
# MySQL: mysql://user:pass@host:3306/db
# PostgreSQL: postgres://user:pass@host:5432/db
DB_URL=registry.db
```

### 5.2 配置加载

```typescript
// src/config.ts
import { randomBytes } from "crypto";

export const config = {
  port: Number(process.env.PORT ?? 3000),
  baseUrl: process.env.BASE_URL ?? `http://localhost:${process.env.PORT ?? 3000}`,
  authTokens: (process.env.AUTH_TOKENS ?? "").split(",").filter(Boolean),
  dataDir: process.env.DATA_DIR ?? "./data",
  db: {
    type: (process.env.DB_TYPE ?? "sqlite") as "sqlite" | "mysql" | "postgresql",
    url: process.env.DB_URL ?? "registry.db",
  },
  maxFileSize: Number(process.env.MAX_FILE_SIZE ?? 100 * 1024 * 1024), // 100MB
  // Web 管理界面认证
  adminUser: process.env.ADMIN_USER ?? "admin",
  adminPass: process.env.ADMIN_PASS ?? "",
  sessionSecret: process.env.SESSION_SECRET ?? randomBytes(32).toString("hex"),
} as const;

export type AppConfig = typeof config;
```

### 5.3 最小启动

```bash
AUTH_TOKENS=my-secret BASE_URL=https://plugins.example.com bun run src/index.ts
```
