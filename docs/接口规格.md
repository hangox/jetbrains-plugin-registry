# API 规格

## 概览

| 路径 | 方法 | 认证 | 用途 |
|------|------|------|------|
| `/updatePlugins.xml` | GET | 无 | JetBrains IDE 拉取插件列表 |
| `/plugins/{pluginId}/{version}` | GET | 无 | JetBrains IDE 下载插件 |
| `/api/plugins` | GET | 无 | 查询插件列表 |
| `/api/plugins` | POST | Bearer Token | 上传插件 |
| `/api/plugins/{pluginId}` | GET | 无 | 查询插件详情（含所有版本） |
| `/api/plugins/{pluginId}` | DELETE | Bearer Token | 删除整个插件 |
| `/api/plugins/{pluginId}/{version}` | DELETE | Bearer Token | 删除指定版本 |
| `/web/login` | POST | 无 | Web 管理界面登录 |
| `/web/logout` | POST | Session | Web 管理界面登出 |
| `/api/health` | GET | 无 | 健康检查 |
| `/api/stats` | GET | 无 | 存储统计 |

## 1. JetBrains Repository 协议接口

这两个接口是 JetBrains IDE 直接调用的，格式必须严格遵循 Custom Plugin Repository 协议。

### 1.1 GET /updatePlugins.xml

IDE 通过此接口发现可用插件。IDE 会自动附加 `build` 参数。

**请求：**

```
GET /updatePlugins.xml?build=IC-241.15989.150
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `build` | string | 否 | IDE 的完整 build number，格式 `{productCode}-{buildNumber}`。不传时返回所有插件的最新版本。 |

**build 参数解析规则：**

IDE 发送的 build number 格式为 `IC-241.15989.150`，服务需要：
1. 去掉产品前缀（`IC-` 部分），取 `241.15989.150`
2. 与 plugin.xml 中的 `since-build` / `until-build` 做范围匹配
3. 版本比较规则：按 `.` 分割为数字段，逐段比较。`*` 通配符匹配任意值。

**版本过滤逻辑：**

```
对于每个 plugin version：
  compatible = (sinceBuild <= requestedBuild) AND
               (untilBuild 为空 OR untilBuild >= requestedBuild)

对于每个 plugin id：
  从所有 compatible 的 versions 中取 version 号最大的那个
```

**响应：**

```
HTTP/1.1 200 OK
Content-Type: application/xml; charset=utf-8
```

```xml
<?xml version="1.0" encoding="UTF-8"?>
<plugins>
  <plugin id="com.example.myplugin"
          url="https://plugins.example.com/plugins/com.example.myplugin/1.2.0"
          version="1.2.0">
    <idea-version since-build="222" until-build="241.*"/>
    <name>My Custom Plugin</name>
    <description><![CDATA[<p>这是一个示例插件。</p>]]></description>
    <change-notes><![CDATA[<ul><li>修复了 Bug</li></ul>]]></change-notes>
    <vendor email="dev@example.com" url="https://example.com">Example Inc</vendor>
  </plugin>
  <plugin id="com.example.another"
          url="https://plugins.example.com/plugins/com.example.another/2.0.0"
          version="2.0.0">
    <idea-version since-build="231"/>
    <name>Another Plugin</name>
  </plugin>
</plugins>
```

**关键约束（来自 JetBrains 文档）：**

- 每个 `<plugin>` 的 `id` 属性在同一个 XML 文件中只能出现一次
- `url` 属性必须是完整的可下载地址
- `version` 属性必须与插件包内 plugin.xml 中的 `<version>` 一致
- `<idea-version>` 的 `since-build` 是必填的，`until-build` 可选

### 1.2 GET /plugins/{pluginId}/{version}

IDE 下载具体的插件包。

**请求：**

```
GET /plugins/com.example.myplugin/1.2.0
```

| 参数 | 位置 | 说明 |
|------|------|------|
| `pluginId` | path | 插件 ID（如 `com.example.myplugin`） |
| `version` | path | 版本号（如 `1.2.0`） |

**响应（成功）：**

```
HTTP/1.1 200 OK
Content-Type: application/zip
Content-Disposition: attachment; filename="com.example.myplugin-1.2.0.zip"
Content-Length: 1234567
Accept-Ranges: bytes

<binary zip data>
```

支持 `Range` 头，允许断点续传（Bun.file() 原生支持 Range 请求）。

**响应（不存在）：**

```
HTTP/1.1 404 Not Found
Content-Type: application/json

{
  "error": "Plugin version not found",
  "pluginId": "com.example.myplugin",
  "version": "9.9.9"
}
```

## 2. 管理 API

### 2.1 POST /api/plugins — 上传插件

**请求：**

```
POST /api/plugins?force=false
Authorization: Bearer <token>
Content-Type: multipart/form-data

------Boundary
Content-Disposition: form-data; name="file"; filename="my-plugin-1.0.0.zip"
Content-Type: application/zip

<binary zip data>
------Boundary--
```

| 参数 | 位置 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| `file` | body (multipart) | file | 是 | - | 插件 ZIP/JAR 文件 |
| `force` | query | boolean | 否 | `false` | 为 true 时覆盖已存在的同版本 |

**响应（成功）：**

```
HTTP/1.1 201 Created
Content-Type: application/json
```

```json
{
  "pluginId": "com.example.myplugin",
  "version": "1.0.0",
  "sinceBuild": "222",
  "untilBuild": "241.*",
  "changeNotes": "<ul><li>Initial release</li></ul>",
  "depends": ["com.intellij.modules.platform", "org.jetbrains.plugins.yaml"],
  "fileName": "com.example.myplugin-1.0.0.zip",
  "fileSize": 1234567,
  "fileSha256": "a1b2c3d4...",
  "createdAt": "2026-02-21T10:00:00Z"
}
```

**错误响应：**

| 状态码 | 场景 | body |
|--------|------|------|
| 400 | ZIP 结构非法，找不到 plugin.xml | `{"error": "Invalid plugin package: no plugin.xml found"}` |
| 400 | plugin.xml 缺少必填字段 | `{"error": "Missing required field: <id>"}` |
| 401 | Token 无效或未提供 | `{"error": "Unauthorized"}` |
| 409 | 同 id+version 已存在且未传 force=true | `{"error": "Version 1.0.0 already exists for com.example.myplugin. Use ?force=true to overwrite."}` |
| 413 | 文件超过大小限制（默认 100MB） | `{"error": "File too large. Max size: 100MB"}` |

### 2.4 DELETE /api/plugins/{pluginId}/{version} — 删除指定版本

**请求：**

```
DELETE /api/plugins/com.example.myplugin/1.0.0
Authorization: Bearer <token>
```

**响应（成功）：**

```
HTTP/1.1 200 OK
Content-Type: application/json

{
  "message": "Deleted com.example.myplugin version 1.0.0",
  "remainingVersions": 2
}
```

如果删除的是该插件的最后一个版本，同时删除 plugins 表记录和磁盘目录。

**响应（不存在）：**

```
HTTP/1.1 404 Not Found

{"error": "Plugin version not found"}
```

### 2.5 DELETE /api/plugins/{pluginId} — 删除整个插件

**请求：**

```
DELETE /api/plugins/com.example.myplugin
Authorization: Bearer <token>
```

**响应：**

```json
{
  "message": "Deleted com.example.myplugin with all 3 versions"
}
```

删除该插件的所有版本文件 + 所有数据库记录 + 磁盘目录。

## 3. Web 认证接口

### 3.1 POST /web/login — 登录

Web 管理界面的登录接口，校验用户名密码后设置 session cookie。

**请求：**

```
POST /web/login
Content-Type: application/x-www-form-urlencoded

username=admin&password=your-password
```

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| `username` | body | string | 是 | 管理员用户名 |
| `password` | body | string | 是 | 管理员密码 |

**响应（成功）：**

```
HTTP/1.1 302 Found
Set-Cookie: session=<signed-token>; HttpOnly; SameSite=Strict; Path=/
Location: /
```

登录成功后设置 httpOnly cookie 并重定向到首页（或用户之前访问的受保护页面）。

**响应（失败）：**

```
HTTP/1.1 302 Found
Location: /web/login?error=Invalid+username+or+password
```

| 状态码 | 场景 | 说明 |
|--------|------|------|
| 302 | 成功 | 设置 cookie 后重定向到首页 |
| 302 | 失败 | 重定向到登录页并附带错误信息 |

### 3.2 POST /web/logout — 登出

清除 session cookie，重定向到登录页。

**请求：**

```
POST /web/logout
Cookie: session=<signed-token>
```

**响应：**

```
HTTP/1.1 302 Found
Set-Cookie: session=; HttpOnly; SameSite=Strict; Path=/; Max-Age=0
Location: /web/login
```

## 5. 认证机制

系统使用两种独立的认证方式，分别服务于不同的使用场景。

### 5.1 API Token 认证（Gradle 插件 / CI）

用于 Gradle 插件、CI/CD 脚本等程序化访问场景。

- 通过 `Authorization: Bearer <token>` 头传递
- 仅 `POST /api/plugins` 和 `DELETE /api/plugins/*` 需要
- 支持配置多个 Token（逗号分隔），用于区分不同环境或 CI

**配置：**

```bash
# 单 Token
AUTH_TOKENS=my-secret-token

# 多 Token（逗号分隔）
AUTH_TOKENS=ci-token-xxx,dev-token-yyy,admin-token-zzz
```

### 5.2 Web Session 认证（浏览器管理界面）

用于浏览器访问 Web 管理界面的认证。

- 用户通过 `/web/login` 页面输入用户名密码登录
- 登录成功后服务端设置 httpOnly cookie（签名 session）
- 后续所有 Web 写操作（上传、删除）通过 cookie 自动认证
- 登出时清除 cookie

**配置：**

```bash
ADMIN_USER=admin              # 管理员用户名，默认 admin
ADMIN_PASS=your-password      # 管理员密码，必填
SESSION_SECRET=random-string  # cookie 签名密钥，默认自动生成
```

### 5.3 无认证接口

以下接口不需要任何认证（IDE 端免配置）：

- `GET /updatePlugins.xml`
- `GET /plugins/{pluginId}/{version}`
- `GET /api/plugins`（只读查询）
- `GET /api/plugins/{pluginId}`（只读查询）
- `GET /api/health`
- `GET /api/stats`
- `GET /web/login`（登录页面）
- `GET /`（首页，只读浏览）

## 6. 通用错误格式

所有错误响应使用统一 JSON 格式：

```json
{
  "error": "错误描述",
  "detail": "可选的详细信息"
}
```

| 状态码 | 含义 |
|--------|------|
| 400 | 请求格式错误（无效 ZIP、缺少字段等） |
| 401 | 未认证或 Token 无效 |
| 404 | 插件或版本不存在 |
| 409 | 版本冲突（已存在） |
| 413 | 文件过大 |
| 500 | 服务内部错误 |

## 7. 路由实现

### 7.1 Repository 路由（JetBrains 协议）

```typescript
// src/routes/repository.ts
import { Hono } from "hono";
import type { PluginService } from "../service/plugin-service";
import type { AppConfig } from "../config";

export function createRepositoryRoutes(service: PluginService, config: AppConfig) {
  const repo = new Hono();

  // IDE 拉取插件列表
  repo.get("/updatePlugins.xml", (c) => {
    const buildParam = c.req.query("build") || null;

    // 去掉产品前缀（如 "IC-241.15989.150" → "241.15989.150"）
    let buildNumber: string | null = null;
    if (buildParam) {
      const dashIndex = buildParam.indexOf("-");
      buildNumber = dashIndex >= 0 ? buildParam.substring(dashIndex + 1) : buildParam;
    }

    const xml = service.getUpdatePluginsXml(buildNumber);
    return c.body(xml, 200, {
      "Content-Type": "application/xml; charset=utf-8",
    });
  });

  // IDE 下载插件文件
  repo.get("/plugins/:pluginId/:version", async (c) => {
    const { pluginId, version } = c.req.param();
    const filePath = service.getPluginFilePath(pluginId, version);

    const file = Bun.file(filePath);
    if (!(await file.exists())) {
      return c.json({
        error: "Plugin version not found",
        pluginId,
        version,
      }, 404);
    }

    return new Response(file, {
      headers: {
        "Content-Type": "application/zip",
        "Content-Disposition": `attachment; filename="${pluginId}-${version}.zip"`,
        "Content-Length": String(file.size),
        "Accept-Ranges": "bytes",
      },
    });
  });

  return repo;
}
```

### 7.2 管理 API 路由

```typescript
// src/routes/api.ts
import { Hono } from "hono";
import { bearerAuth } from "hono/bearer-auth";
import type { PluginService } from "../service/plugin-service";
import type { AppConfig } from "../config";

export function createApiRoutes(service: PluginService, config: AppConfig) {
  const api = new Hono();

  // Bearer Token 认证中间件（仅保护写操作）
  const auth = bearerAuth({
    verifyToken: (token) => config.authTokens.includes(token),
  });

  // ── 只读接口 ──

  // 查询插件列表
  api.get("/plugins", (c) => {
    const query = c.req.query("query") || null;
    const page = Math.max(1, Number(c.req.query("page") ?? 1));
    const pageSize = Math.min(100, Math.max(1, Number(c.req.query("pageSize") ?? 20)));
    const result = service.listPlugins(query, page, pageSize);
    return c.json(result);
  });

  // 查询插件详情
  api.get("/plugins/:pluginId", (c) => {
    const pluginId = c.req.param("pluginId");
    const plugin = service.getPlugin(pluginId);
    if (!plugin) {
      return c.json({ error: "Plugin not found" }, 404);
    }
    return c.json(plugin);
  });

  // 健康检查
  api.get("/health", (c) => {
    return c.json({
      status: "ok",
      uptime: service.getUptime(),
      database: "connected",
      version: "1.0.0",
    });
  });

  // 存储统计
  api.get("/stats", (c) => {
    const stats = service.getStats();
    const recentUploads = service.getRecentUploads(10);
    return c.json({
      ...stats,
      totalStorageHuman: formatBytes(stats.totalStorageBytes),
      recentUploads,
    });
  });

  // ── 写操作（需认证） ──

  // 上传插件
  api.post("/plugins", auth, async (c) => {
    const body = await c.req.parseBody();
    const file = body["file"] as File;
    const force = c.req.query("force") === "true";

    if (!file || file.size === 0) {
      return c.json({ error: "No file provided" }, 400);
    }

    if (file.size > config.maxFileSize) {
      const maxMb = Math.round(config.maxFileSize / 1024 / 1024);
      return c.json({ error: `File too large. Max size: ${maxMb}MB` }, 413);
    }

    try {
      const result = await service.upload(file, force);
      return c.json(result, 201);
    } catch (e) {
      if (e instanceof Error && e.message.includes("already exists")) {
        return c.json({ error: e.message }, 409);
      }
      if (e instanceof Error && e.name === "InvalidPluginError") {
        return c.json({ error: e.message }, 400);
      }
      throw e;
    }
  });

  // 删除指定版本
  api.delete("/plugins/:pluginId/:version", auth, async (c) => {
    const { pluginId, version } = c.req.param();
    const existing = service.getPlugin(pluginId);
    if (!existing || !existing.versions.find((v) => v.version === version)) {
      return c.json({ error: "Plugin version not found" }, 404);
    }

    const remaining = await service.deleteVersion(pluginId, version);
    return c.json({
      message: `Deleted ${pluginId} version ${version}`,
      remainingVersions: remaining,
    });
  });

  // 删除整个插件
  api.delete("/plugins/:pluginId", auth, async (c) => {
    const pluginId = c.req.param("pluginId");
    const existing = service.getPlugin(pluginId);
    if (!existing) {
      return c.json({ error: "Plugin not found" }, 404);
    }

    const versionCount = existing.versions.length;
    await service.deletePlugin(pluginId);
    return c.json({
      message: `Deleted ${pluginId} with all ${versionCount} versions`,
    });
  });

  return api;
}

function formatBytes(bytes: number): string {
  if (bytes === 0) return "0 B";
  const units = ["B", "KB", "MB", "GB"];
  const i = Math.floor(Math.log(bytes) / Math.log(1024));
  return `${(bytes / Math.pow(1024, i)).toFixed(1)} ${units[i]}`;
}
```

### 7.3 XML 生成

```typescript
// src/lib/xml.ts
import type { CompatiblePlugin } from "../repository/types";

/**
 * 生成 JetBrains Custom Plugin Repository 格式的 XML。
 * 每个 pluginId 只返回一个最兼容的版本。
 */
export function generateUpdatePluginsXml(plugins: CompatiblePlugin[]): string {
  const entries = plugins.map((p) => {
    let xml = `  <plugin id="${escapeXml(p.id)}"
          url="${escapeXml(p.downloadUrl)}"
          version="${escapeXml(p.version)}">
    <idea-version since-build="${escapeXml(p.sinceBuild)}"${
      p.untilBuild ? ` until-build="${escapeXml(p.untilBuild)}"` : ""
    }/>
    <name>${escapeXml(p.name)}</name>`;

    if (p.description) {
      xml += `\n    <description><![CDATA[${p.description}]]></description>`;
    }
    if (p.changeNotes) {
      xml += `\n    <change-notes><![CDATA[${p.changeNotes}]]></change-notes>`;
    }
    if (p.vendor) {
      const attrs: string[] = [];
      if (p.vendor.email) attrs.push(`email="${escapeXml(p.vendor.email)}"`);
      if (p.vendor.url) attrs.push(`url="${escapeXml(p.vendor.url)}"`);
      xml += `\n    <vendor${attrs.length ? " " + attrs.join(" ") : ""}>${escapeXml(p.vendor.name)}</vendor>`;
    }

    xml += `\n  </plugin>`;
    return xml;
  });

  return `<?xml version="1.0" encoding="UTF-8"?>\n<plugins>\n${entries.join("\n")}\n</plugins>`;
}

function escapeXml(str: string): string {
  return str
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;");
}
```
